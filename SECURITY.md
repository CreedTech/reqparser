# Security Policy

Thank you for helping keep `reqparser` and its users safe.

## Supported versions

This project is still in early development. Security fixes will be applied to the latest version on the default branch.

## Reporting a vulnerability

Please do **not** open a public issue for a suspected security vulnerability.

Instead:

1. Email the maintainer privately if a security contact is available.
2. Or open a GitHub private vulnerability report for this repository.
3. Include clear reproduction steps, impact, and any suggested remediation if possible.

You will receive an acknowledgement as soon as practical, and we will work on validation, remediation, and coordinated disclosure.

## Scope

Please report issues related to:

- secret leakage
- unsafe request redaction
- dependency vulnerabilities with real impact on this project
- command injection or code execution risks
- malformed parsing that could create dangerous generated output

General bugs, feature requests, and parser edge cases without security impact should go through normal GitHub issues.
